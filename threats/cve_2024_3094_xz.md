# CVE-2024-3094 — XZ Utils Backdoor (post-exploitation behaviour)

A backdoor was implanted in the upstream `xz-utils` library (`liblzma`),
loaded into the `sshd` process on affected Linux systems. The backdoor
enables an authenticated remote attacker to bypass SSH authentication
when presenting a specific signed payload, after which they execute
commands as `root`.

This detection focuses on observable post-exploitation behaviour visible
in the Counterspell data model (`cs:auth`, `cs:process`), not on the
binary supply-chain compromise itself.

Characteristics to detect:

- A successful authentication via `app=ssh` to a host where, within the
  next 60 seconds, a `cs:process` event is observed with
  `parent_process=sshd` and `process_name` in (`bash`, `sh`, `python`,
  `powershell.exe`) running with `user=root` or another privileged account.
- The same `src_ip` appears in `cs:auth` only once or twice (not a
  brute-force pattern) — this is *clean* authentication followed by
  immediate privileged execution.
- Optionally: outbound `cs:network` from the same host within the next
  5 minutes to an external IP on a non-standard port (the operator
  callback).

MITRE ATT&CK:
- T1190 — Exploit Public-Facing Application
- T1059 — Command and Scripting Interpreter
- T1078 — Valid Accounts (the SSH bypass presents as a valid auth)

Data sources needed: `cs:auth` (action, app, src_ip, user, host, _time),
`cs:process` (parent_process, process_name, user, host, _time),
optionally `cs:network` (src_ip, dest_ip, dest_port, bytes_out, _time).
