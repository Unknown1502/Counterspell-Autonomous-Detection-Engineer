# Changelog

All notable changes to Counterspell are documented here. The format is based on
[Keep a Changelog](https://keepachangelog.com/en/1.1.0/) and this project
adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- **Extended threat portfolio.** Threat fixtures expanded from 3 to 9,
  covering 13+ MITRE techniques across all three sourcetypes:
  `t1059_powershell_encoded.md` (LOLBin abuse),
  `t1071_c2_beacon.md` (C2 beaconing — interval regularity, not volume),
  `t1003_lsass_dump.md` (credential dumping),
  `t1078_impossible_travel.md` (auth from disparate geographies),
  `cve_2024_23897_jenkins.md` (Jenkins CLI file read CVE),
  `t1486_ransomware_burst.md` (ransomware detonation + double-extortion).
  See [`threats/README.md`](threats/README.md) for the headline-demo set vs
  the extended portfolio driving the Navigator coverage map.
- **Data generator enhanced with per-threat noise floors.** Each new
  threat in the extended portfolio now has a matching benign baseline so
  the FP curve actually has room to drop:
  - Periodic CDN/telemetry "beacons" (4 fixed src→dest profiles, 300–900 s
    intervals with jitter) — baseline for T1071.001 detection tuning
  - Daily 18–35-event backup-tool process bursts on a designated host
    (Veeam, wbengine, ConnectWise, Acronis) — baseline for T1486
  - ~5% of cs:process events are encoded PowerShell from automation
    parents (services.exe, ccmexec.exe, taskeng.exe, etc.) — baseline for
    T1059.001
  - ~0.2% benign procdump runs targeting hung admin services — baseline
    for T1003.001
  - Build-host activity (`jenkins-01`, `build-02`) with `svc_jenkins`
    running `java -jar jenkins-cli.jar ...` — baseline for CVE-2024-23897
  - 5 "roaming" users authenticate from a mix of internal / `192.168.x.x`
    / external IPs ~40% of the time — baseline for T1078 impossible travel
- **`COUNTERSPELL_SEED` env var** makes the baseline fully reproducible
  between recordings — same seed = same FP curve every take.
- **5 new data acceptance checks** (`scripts/check_data_acceptance.py`)
  validate that each new noise floor actually got seeded: encoded-PS,
  periodic beacons, build-host activity, daily backup bursts, roaming
  users. Total checks: 10.
- **Splunk Enterprise Security integration.** Every deployed saved search
  now carries notable-event and Risk-Based Alerting metadata
  (`actions.notable`, `action.risk`, risk objects keyed on the design's
  entity fields), wired in [`agents/deployer.py`](src/counterspell/agents/deployer.py).
- **MITRE ATT&CK Navigator layer export** — `scripts/export_navigator_layer.py`
  reads run logs and emits a Navigator-compatible `coverage.json` (scored,
  with FP-curve comments per technique).
- **Multi-threat batch runner** — `scripts/run_all_demos.py` runs every
  threat fixture in sequence with a Rich coverage summary.
- **Deduplication** — orchestrator checks the `counterspell_runbook` KV
  collection for an existing detection covering the same MITRE technique
  before proceeding, and emits a `duplicate` event on overlap.
- **Run-log persistence** — every run appends one JSON line to
  `runs/<YYYY-MM-DD>.jsonl`, feeding the Navigator export.
- **Dashboard upgrade** — three big-number panels (latest FP, TP caught,
  iterations), explicit "first vs final SPL" side-by-side, MITRE coverage
  bar chart, ES action columns surfaced in the deployed-search table.
- **Data acceptance script** — `scripts/check_data_acceptance.py`
  automates the manual checklist in [`docs/06_DATA_MODEL.md`](docs/06_DATA_MODEL.md).
- **AppInspect-ready icons** — `splunk_app/static/appIcon.png` and
  `appIcon_2x.png`, generated by `scripts/generate_icons.py`.
- **CI** — GitHub Actions workflow at `.github/workflows/tests.yml` runs
  the 41-test suite on every push.

### Changed
- `Orchestrator.run()` now accepts an optional `runs_dir` parameter for
  custom run-log destinations; default is `<repo>/runs/`.
- `Deployer.deploy()` now accepts an optional `design` parameter to
  populate ES metadata. Backwards-compatible: passing only `(doc, spl)`
  still works, just without ES integration.
- Dashboard panels reworked to use a single shared base search so the
  loop is not re-invoked per panel.

## [0.1.0] — 2026-05-29

### Added
- Initial Counterspell agent runtime: Architect, Translator, Red-team,
  Validator, Deployer.
- Orchestrator with iteration cap and human-approval gate.
- Splunk SDK + MCP JSON-RPC clients with transparent SDK fallback.
- OpenAI-compatible LLM client with JSON-validate-and-retry.
- Synthetic data generator (~30 days of cs:auth, cs:process, cs:network
  with deliberate benign noise for FP-curve room).
- Threat fixtures: `t1048_exfil.md`, `t1110_bruteforce.md`,
  `cve_2024_3094_xz.md`.
- CLI demo (`scripts/run_demo.py`) with Rich progress output.
- Splunk app: custom search command + dashboard + KV runbook lookup +
  ES correlation-search tagging.
- 41 unit tests covering Validator TP/FP logic, LLM JSON parsing,
  MCP response shapes, Translator routing, Orchestrator control flow,
  and schema invariants.
